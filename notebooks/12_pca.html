PCA — Dimensionality Reduction

Question: What structure exists in correlated features?Output: PC scores + 2D projectionDataset: customers.csv (800 rows)Use case: Feature compression · 2D customer map · Noise reduction before clustering

Pipeline walkthrough — analyst workflow steps, pre-populated outputs, and business interpretation at each stage.

Stage 01

When PCA Is the Right Tool

PCA (Principal Component Analysis) reduces many correlated features into a smaller set of uncorrelated components. Use it when you have too many features, when features are highly correlated, or when you need a 2D visualisation of high-dimensional data.

  • Identify the business problem PCA does not produce labels or predictions by itself. It is a preprocessing step or visualisation tool. Common uses: compress correlated features before another model; visualise customer data in 2D; reduce noise before clustering.PCA is not a model — it is a lens. You still need to interpret the components in business terms.
  • Check whether PCA is appropriate PCA works well when features are correlated. If features are independent, PCA will not compress them — each component will explain roughly equal variance, and you gain nothing. Run a correlation heatmap first.
  • Decide on target variance explained Retain enough components to explain 80–95% of variance. For visualisation: keep 2-3 components. For preprocessing before a model: keep components explaining 90% of variance.

Stage 02

Fit PCA & Interpret Variance

Fit PCA on standardised features and read the explained variance plot to choose how many components to retain.

  • Standardise features before PCA Mandatory — PCA is scale-sensitive. Without standardisation, high-variance features dominate all components.
  • Fit PCA(n_components=8) to see full variance spectrum Fit with all components first. Read the cumulative explained variance to decide how many to retain.
  • Plot scree plot (explained variance per component) Bar chart showing % variance per component. The 'elbow' suggests where additional components stop contributing meaningful variance.
  • Inspect component loadings Each PC is a weighted combination of original features. The loading matrix shows which features dominate each PC — this is how you interpret PCs in business terms.

What this tells you
  • PC1 captures the 'depth' of the customer relationship: long-tenured, high-value, engaged customers have high PC1 scores. Low PC1 = at-risk profile.
  • PC2 captures 'friction intensity': customers with high support call rates and older age have high PC2 scores.
  • Two PCs explain 57% of variance and have interpretable business meaning — sufficient for a 2D visualisation. Use 5 PCs for preprocessing before a churn model.
PCA components must be namedA PCA deliverable is not a table of loadings — it is named components with business meaning. 'PC1 = Customer Depth' and 'PC2 = Service Friction' are usable. 'Principal Component 1' is not. If you cannot name a component, the variance it captures may not be business-meaningful.
Stage 03

2D Visualisation

Project all customers into 2D (PC1 × PC2 space) and overlay cluster labels or churn flags for insight.

  • Transform all customers to PC scores Apply fitted PCA to the customer dataset. Each customer now has a (PC1, PC2) coordinate.
  • Scatter plot coloured by churn label Customers who churned in one colour, retained in another. Well-separated clouds = PCA has captured the churn signal.
  • Overlay K-Means cluster assignments Check if K-Means clusters correspond to regions in PC space — this validates that clusters are genuinely distinct in the underlying data.

Output: QA Checklist Checklist
Good
  • Features standardised before PCA
  • Number of components chosen by cumulative variance (not arbitrary)
  • Components named in business terms
  • Scree plot shows clear elbow
  • 2D projection coloured by outcome/segment to validate signal
Red flags
  • PCA applied without standardisation (scale-dominated)
  • All 8 components retained (no dimensionality reduction)
  • Components labelled as PC1, PC2 with no interpretation
  • PCA used as a standalone model rather than preprocessing or visualisation layer
