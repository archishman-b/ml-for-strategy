Stage 01

When Neural Networks Are — and Are Not — the Right Choice

Neural networks are powerful but demanding. For tabular business data (<100K rows, <50 features), tree-based models (GBM) almost always match or outperform neural networks with far less tuning effort. Know when to reach for this tool.

  • Check dataset size first Neural networks are data-hungry. Rule of thumb: < 10K rows → use GBM or LR. 10K–100K rows → GBM usually wins. > 100K rows → neural networks become competitive. For image, text, or sequence data at any scale: neural networks are the default.On the 800-row customers.csv dataset, a neural network will almost certainly underperform LightGBM. The workflow below uses a larger synthetic dataset for demonstration.
  • Understand what you are building Feedforward network (MLP): the standard. Embedding layers: for high-cardinality categoricals (e.g. 10,000 SKUs). Recurrent (LSTM/GRU): for sequential data. Transformers: for text. For tabular strategy work, MLP with embeddings is the relevant architecture.
  • Establish GBM baseline first Always run LightGBM first. If GBM AUC is within 0.01 of the best neural network, use GBM — it is faster to train, easier to interpret, and requires less infrastructure.

Stage 02

Build & Train a Feedforward Network

For tabular data, the standard architecture is a feedforward network (MLP): input layer → 2-3 hidden layers with ReLU → dropout for regularisation → sigmoid output for binary classification.

  • Define the architecture Input layer: one node per feature (20 features → 20 inputs). Hidden layers: 2 layers × 128 nodes with ReLU activation. Dropout: 0.3 (drops 30% of nodes each training step to prevent overfitting). Output: single sigmoid node → P(churn).Rule of thumb for hidden layer sizing: start with 2× the number of input features. Wider is usually better than deeper for tabular data.
  • Choose optimiser and loss Adam optimiser with lr=0.001 (adaptive learning rate — standard default). Binary cross-entropy loss for binary classification. Batch size=256 for stability on 50K dataset.
  • Set up early stopping callback Stop training when validation AUC stops improving for 10 epochs. Prevents overfitting to the training set.

Neural networks require production infrastructure that GBM does notDeploying a neural network requires: a serving framework (TensorFlow Serving, ONNX), GPU instances for batch scoring at scale, model versioning infrastructure, and more complex drift monitoring. For strategy analytics work, this overhead is rarely justified unless the AUC gap is > 2-3 points. Use GBM as the default.
Stage 03

Embedding Layers for High-Cardinality Categoricals

The one scenario where neural networks genuinely outperform GBM on tabular data: when you have high-cardinality categorical features (e.g., 10,000 SKUs, 5,000 zip codes). Embedding layers learn a dense vector representation for each category.

  • Identify high-cardinality features Standard one-hot encoding of 10,000 SKUs creates 10,000 sparse columns — impractical for GBM and impossible for linear models. Embedding maps each SKU to a dense vector of 16-32 dimensions.Rule of thumb: embedding dimension = min(50, (cardinality + 1) // 2).
  • Build embedding layer per high-cardinality feature Each categorical feature gets its own embedding lookup table. The embeddings are learned jointly with the rest of the network — they capture similarity between categories implicitly.
  • Interpret embeddings After training, SKUs with similar embeddings have similar customer behaviour patterns. This can be used independently as a product similarity metric.

Output: QA Checklist Checklist
Good
  • GBM baseline established and compared
  • Early stopping used — training log shows plateau before stopping
  • Val loss and train loss both declining (no overfitting)
  • Dropout applied (not just L2 regularisation alone)
  • Embedding layers used for high-cardinality features (>100 categories)
Red flags
  • No GBM baseline — neural network chosen first
  • Training reported only on train loss (not validation)
  • Neural network deployed for a 1,000-row dataset
  • No early stopping — fixed number of epochs
  • AUC advantage over GBM < 0.01 — does not justify infrastructure cost
