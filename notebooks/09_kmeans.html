Notebook 3.1

K-Means Clustering

Question: Who are my customer segments?Output: Cluster labelDataset: customers.csv (800 rows)Use case: Customer value tiering · Micro-market prioritisation

Pipeline walkthrough — analyst workflow steps, pre-populated outputs, and business interpretation at each stage.

Stage 01

Feature Scaling & K Selection

K-Means partitions customers into K groups by minimising within-cluster variance. Two non-negotiable steps before fitting: scale features and select K.

  • Scale all numeric features first K-Means uses Euclidean distance. An unscaled revenue feature (range 0–100K) will completely dominate an unscaled age feature (range 18–80).This is the most common K-Means mistake. Always StandardScaler before fitting.
  • Select K with elbow method and silhouette score Fit K-Means for K=2 to 10. Plot inertia (within-cluster sum of squares) vs K. The 'elbow' point is the candidate K. Cross-validate with silhouette score (higher is better, range -1 to 1).
  • Run multiple initialisations for stability K-Means random initialisation can produce different results. Use K-Means++ initialisation (default in sklearn) and run n_init=10 times, keeping the best.

Stage 02

Fit & Profile Clusters

Fit the final model, assign every customer to a cluster, and build cluster profiles — the actionable output.

  • Fit KMeans(n_clusters=5, init='k-means++', n_init=10) train on all customers. Each customer gets a cluster assignment (0-4).
  • Build centroid profile table For each cluster, compute mean of each original feature. Translate back to original units (inverse-standardise) for human readability.
  • Name each cluster Clusters must be named in business terms. If you cannot give a cluster a meaningful name, the model needs to be re-run or features reconsidered.A cluster named 'Cluster 3' is not a deliverable. 'At-Risk New Customers' is.
  • Assign cluster sizes and validate business relevance A cluster containing <1% of customers is probably noise. A cluster containing 70% of customers is probably too broad.

What this tells you
  • At-Risk New (Cluster 2) has 48% churn rate and only 89 customers — but these 89 are the highest-priority intervention target.
  • Champions (Cluster 0) at 7% churn rate — minimal intervention needed. Over-investing in retention for this group is wasted spend.
  • The cluster profile table IS the deliverable for a retention strategy session. Each row becomes a distinct workstream.
Clusters are a lens, not ground truthK-Means clusters are an analytical construct — they are not immutable customer categories. Customers move between clusters as their behaviour changes. Re-segment quarterly and track cluster migration rates.
Stage 03

QA Checklist

  • Verify features were scaled Ask: was StandardScaler applied before fitting? Unscaled features = meaningless clusters.
  • Business validation test Can every cluster be named in business terms? Can you define a distinct action for each? If not, the K is wrong or the features are wrong.
  • Stability test Re-run with 3 different random seeds. If cluster assignments change dramatically, the model is unstable — increase n_init or try Hierarchical Clustering.
Output: QA Checklist
Good
  • Features standardised before K-Means
  • K chosen via elbow + silhouette (both checked)
  • Every cluster named in business terms
  • Distinct action defined per cluster
  • Stability confirmed across multiple seeds
Red flags
  • Unscaled features fed to K-Means
  • K chosen arbitrarily ('we want 4 segments')
  • Cluster 'profiles' are just number tables with no business labels
  • One cluster contains >60% of customers (too broad)
  • Model re-run once — no stability check
