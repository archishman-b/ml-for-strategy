Stage 01

Text Classification Setup

Naive Bayes shines on text and sparse high-dimensional data. For ticket routing, we vectorise text with TF-IDF then apply Multinomial NB.

  • Choose the right NB variant Gaussian NB: numeric features. Multinomial NB: word counts (text). Bernoulli NB: binary presence/absence. For ticket text → TF-IDF + Multinomial NB is standard.Choosing the wrong variant is the most common setup error.
  • Apply TF-IDF vectorisation Convert ticket text to term-frequency × inverse-document-frequency matrix. Common words auto-downweighted. min_df=2 removes terms appearing in only one ticket.
  • No feature scaling needed NB learns class-conditional probabilities from feature statistics. Scale does not affect relative probability estimates.

Stage 02

Train & Evaluate

Fit, score, and read the classification report per urgency class.

  • Fit MultinomialNB(alpha=1.0) Laplace smoothing (alpha=1) prevents zero-probability for unseen words — important when vocabulary has rare terms.
  • Evaluate per-class performance For ticket routing, recall on High priority is most important — missing an urgent ticket is more costly than a false alarm.

What this tells you
  • 81% accuracy vs 44% baseline — NB is a viable routing layer but not reliable enough for High priority alone.
  • Most discriminating features (outage, broken) are intuitively correct — the vocabulary check validates the model.
  • Use as first-pass router: auto-assign Low/Medium, flag potential High for human review.
The independence assumption in practiceNB assumes 'outage' and 'cannot access' are independent given the class — they're not. But the ranking still works well enough for routing. Miscalibrated probabilities don't matter when you're only ranking.
Stage 03

QA Checklist

Sign-off questions for NB classification.

  • Compare AUC vs LR on same features TF-IDF + LR often outperforms NB. Gap < 2 points: keep NB for speed. Gap > 5 points: switch to LR.
  • Verify alpha smoothing alpha=0 causes P=0 for unseen words, breaking the model. Always alpha >= 0.1.
Output: NB vs LR Console
Multinomial NB (alpha=1.0): accuracy=0.81, macro F1=0.78, train: 0.02s Logistic Regression: accuracy=0.84, macro F1=0.81, train: 0.18s Gap: 3 points macro F1. LR better but NB acceptable for real-time routing. Decision: NB for first-pass routing (speed), LR for nightly batch scoring.
Output: QA Checklist
Good output
  • NB variant matches feature type (Multinomial for counts)
  • alpha smoothing applied
  • AUC compared against LR baseline
  • Top terms checked for interpretability
Red flags
  • alpha=0 (zero probabilities on unseen words)
  • Gaussian NB applied to text counts (wrong variant)
  • Calibration relied on for threshold-based decisions (NB probabilities are often miscalibrated)
  • Precision/recall shown only for majority class
