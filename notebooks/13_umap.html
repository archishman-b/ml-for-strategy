Pipeline walkthrough — analyst workflow steps, pre-populated outputs, and business interpretation at each stage.

Stage 01

UMAP vs PCA — When to Use Each

UMAP (Uniform Manifold Approximation and Projection) is a nonlinear dimensionality reduction technique. It preserves local structure — similar customers in high-dimensional space will be close together in 2D UMAP space. PCA preserves global variance; UMAP preserves neighbourhood relationships.

  • Choose UMAP when nonlinear structure exists PCA flattens everything linearly. UMAP will reveal clusters, manifolds, and local groupings that PCA would blur together. For customer data with distinct behavioural segments: UMAP often produces cleaner separations.UMAP is a visualisation tool primarily. It is not deterministic — two runs produce different layouts. Do not use raw UMAP coordinates as model features.
  • Understand the two key parameters n_neighbors: controls local vs global structure balance. Low (5–10): preserves fine local structure, may fragment large clusters. High (30–50): preserves global topology, smoother. min_dist: how tightly points are packed. Low (0.0–0.1): dense clumps. High (0.5+): spread out.
  • Install and import UMAP umap-learn library. Always set random_state for reproducibility.

Stage 02

Fit & Visualise

Fit UMAP and interpret the 2D map. The key question: do similar customers cluster together in UMAP space?

  • Standardise features first Same requirement as PCA. UMAP uses distances — scale matters.
  • Fit UMAP and extract 2D coordinates Each customer maps to (x, y) in UMAP space. Nearby points = similar customers in the original 8-dimensional space.
  • Colour-code by known labels Overlay churn label, K-Means cluster, tier, or region. If UMAP has captured the right structure, churned customers should cluster together.

What this tells you
  • UMAP confirms the K-Means clustering result — At-Risk New customers form a genuinely tight cluster in feature space (compact region in UMAP), not just a K-Means artefact.
  • The diffuse central cloud confirms that mid-tier customers (Clusters 1, 3, 4) genuinely overlap in behaviour — the K-Means boundaries between them are soft.
  • UMAP is best used as a validation and EDA tool: 'does the structure we assumed exist actually exist?' before committing to a segmentation strategy.
Never use UMAP coordinates as model featuresUMAP coordinates change every time you re-fit (even with fixed random seed on different data batches). They are a visualisation output, not stable feature representations. If you need compressed feature inputs for a downstream model, use PCA (deterministic) or an autoencoder (stable across batches).
Stage 03

QA

  • Run with multiple n_neighbors values n_neighbors=5, 15, 30, 50. The gross structure should be consistent — if results change dramatically, the data may not have stable local structure.
  • Validate against K-Means or known labels UMAP structure should align with known segment membership if those segments are genuine.
Output: QA Checklist Checklist
Good
  • Features standardised
  • random_state set
  • Multiple n_neighbors values tested
  • UMAP structure validated against K-Means or known labels
  • Used as visualisation/EDA only — not as model features
Red flags
  • UMAP coordinates used as input features to another model
  • Single n_neighbors value without sensitivity check
  • Clusters visible only at one n_neighbors setting
  • UMAP presented as the final segmentation model (it is a visualisation)
